# smart-mqtt 产品介绍

smart-mqtt 是 smartboot 开源组织推出的商业化 MQTT Broker 产品，专为拥有上万级设备连接量的企业级物联网场景设计。

![](./img/framework.svg)


## 关于 smart-mqtt

smart-mqtt 是 smartboot 组织内首个尝试走商业化路线的作品。

虽然我们知道用户更喜欢免费的开源产品，决定探索商业化路线势必会遭受一些非议。更何况 smart-mqtt 是那么的年轻，距离成熟的商业化产品还有很长一段路要走。

但是多年开源经历，我们看到了很多开源作者正是因为没能从开源中获得可持续的收益，最终因生活所迫无奈选择终止维护开源项目。

:::tip[开源初心]
**开源不易，我们始于热爱，敬于技术，更要忠于生活**。
:::

从技术角度而言，一款能够承载海量设备互通互联的产品，一定是具备相当高的业务价值。打造这样一款产品，要么背后有大厂支持为研发成本买单；要么依靠市场的正向反馈使其可持续迭代发展下去，而 smart-mqtt 正是后者。

> 好的产品，本身就该成为好的商品；一个有价值的产品，理当成为一款有价格的商品。


## 核心优势

### 🚀 超高性能

smart-mqtt 底层通信采用自研的框架 [smart-socket](https://gitee.com/smartboot/smart-socket)，并在 [TechEmpower](https://www.techempower.com/benchmarks/) 用数据证明了 Java 的性能并不低于任何的计算机语言。

- 单机支持 **百万级连接**
- 消息吞吐量可达 **1000万/秒**（QoS0）
- v1.5.1 版本包体积不足 **10MB**

### 🔧 开放架构

基于事件总线的架构模式：
- 可将非 MQTT 的南向协议适配融入 smart-mqtt
- 可将 MQTT 消息桥接至任意的北向服务
- 支持插件化扩展，功能按需组合

### ☕ Java 生态

在当下以 Java 为主流开发语言的环境下：
- 天然适合绝大多数从业者
- 易于集成和二次开发
- 完善的 Java 技术栈支持

### 🔄 可替代性

smart-mqtt 完全遵循 MQTT 协议规范：
- 无需担心被本产品捆绑
- 任何时候可快速切换至其他同类产品
- 标准协议，平滑迁移

### 🇨🇳 纯国产血统

符合信创场景需求：
- 底层通信：自研 [smart-socket](https://gitee.com/smartboot/smart-socket)
- HTTP 服务：自研 [Feat](https://smartboot.tech/feat)
- 代码可控，安全可靠

## 管理平台

smart-mqtt 通过内置的 **enterprise-plugin** 插件，作为产品的基础性核心组件，提供功能完善的 Web 管理控制台和强大的插件生命周期管理能力。

服务正常启动后可通过浏览器访问 http://localhost:18083/ （IP根据实际情况调整）进入平台的登录界面。
![](./img/login.png)


管理平台基于 HTTP 服务提供 RESTful API，前端采用 Shadcn UI 框架构建，主要功能模块包括：

| 功能模块 | 说明 |
|----------|------|
| **用户认证** | 基于 Session 的用户登录，支持用户名+密码认证 |
| **License 管理** | 授权验证、License 导入、状态展示 |
| **Broker 管理** | 集群节点列表查询 |
| **连接管理** | MQTT 客户端连接查询、强制断开连接 |
| **Topic 管理** | 订阅关系查询、Topic 统计查询 |
| **插件管理** | 插件列表、启停控制、配置管理、插件市场、本地上传安装 |
| **系统设置** | 用户管理、系统参数配置 |

### Dashboard

仪表盘是管理平台的可视化数据展示中心，提供 Broker 运行状态的实时监控能力：
![](./img/dashboard.png)
- **连接指标**：实时展示当前活跃连接数、历史连接峰值、连接速率等关键指标
- **消息指标**：监控消息收发速率、吞吐量、消息积压情况等
- **系统资源**：展示服务器 CPU、内存、磁盘等基础资源使用情况
- **趋势图表**：支持按时间维度展示指标变化趋势，便于分析业务规律

### 连接管理

![](./img/connect.png)
提供 MQTT 客户端连接的实时监控和管理能力：

- **连接列表查询**：支持分页查询，可按 ClientID 模糊搜索

### 插件管理

在物联网业务快速迭代的今天，企业需要的不是一个功能固化的 MQTT Broker，而是一个能够随业务成长、灵活适配多样场景的可扩展平台。smart-mqtt 的插件管理体系，正是为此而生。

#### 为什么需要插件管理？

**场景一：业务快速变化，功能需求难以预估**

物联网项目从 POC 到量产，从单一设备接入到多协议融合，业务需求始终处于动态变化中。传统软件"一刀切"的功能打包方式，往往导致企业为不需要的功能买单，或关键时刻发现缺少必备能力。

**场景二：系统越复杂，运维越脆弱**

每引入一个新功能，都可能带来新的配置项、新的故障点和新的学习成本。运维团队被迫在"功能缺失"和"系统臃肿"之间艰难取舍。

**场景三：私有化部署的定制困境**

企业自研的功能模块、内部系统的对接需求、行业合规的特殊要求——这些高度定制化的需求，很难在通用产品中得到满足，而二次开发又意味着高昂的成本和长期的维护负担。

#### 插件管理能为您带来什么？

**像搭积木一样构建您的 MQTT 平台**

无需关心底层实现，通过可视化的插件控制台，按需启用所需能力。需要 WebSocket 支持？一键开启。需要对接 Redis？勾选安装。业务调整需要停用某项功能？随时关闭，资源零浪费。

- **连接层**：WebSocket、SSL/TLS、多协议网关
- **安全层**：简单认证、企业级鉴权、ACL 权限控制
- **数据层**：MySQL 持久化、Redis 桥接、消息队列对接
- **架构层**：集群扩展、会话保持、监控告警

**变更不停机，运维更从容**

插件的热插拔能力让功能变更不再等于服务中断。配置更新实时生效，版本升级无缝切换，让您的运维团队从"变更窗口焦虑"中解放出来，专注于更有价值的业务优化。

**您的专属功能，也能被优雅管理**

基于标准化接口开发的私有插件，同样可以享受平台提供的生命周期管理、配置可视化、运行监控等全套能力。企业不用再为自研功能单独建设管理工具，真正实现"一处管理，全局掌控"。

**持续进化的能力版图**

官方插件库持续覆盖物联网核心场景，从车联网到工业互联网，从智能家居到智慧城市，成熟方案开箱即用。同时，活跃的开发者生态让创新功能触手可及。

![已安装的插件列表](./img/plugin_1.png)

> **一句话总结**：插件管理让 smart-mqtt 从一个标准化的 MQTT Broker，进化为可随业务需求灵活演进的物联网通信平台，让技术真正服务于业务，而非束缚业务。

#### 插件发现与获取

由 smart-mqtt 官方提供的插件，都以内置在产品中。同时 smart-mqtt 还提供两种插件集成方式，以满足不同场景需求：

1. **本地上传**
支持上传自定义开发的 JAR 包进行安装，满足私有化部署和二次开发需求
![安装本地插件](./img/plugin_2.png)

2. **插件市场**
连接远程官方插件仓库，浏览、搜索并下载已发布的插件
![安装本地插件](./img/plugin_3.png)

## 成本估算

云产品的采购成本几乎同连接量成线性增长关系，而 smart-mqtt 本就存在性能溢出的优势，所以**连接量越大，性价比越高**。

| 规格 | SaaS 云产品 | smart-mqtt |
|------|-------------|------------|
| 1000连接/1000TPS/1000订阅 | 1.7万/年 | 3000~5000 + 授权费 |
| 50000连接/20000TPS/1000订阅 | 37万/年 | 5000~10000 + 授权费 |
| 100万连接/20万TPS/100万订阅 | 376.8万/年 | ~10万 + 授权费 |

:::tip[数据来源]
- [阿里云 IoT](https://www.aliyun.com/product/mq4iot)
- [华为云 IoT](https://www.huaweicloud.com/pricing.html#/iothub)
:::

## 适用场景

- **物联网平台**：海量设备接入、实时数据通信
- **即时通讯 (IM)**：高并发消息推送、实时聊天
- **车联网**：车辆实时监控、指令下发
- **工业互联网**：设备监控、数据采集
- **智能家居**：多设备联动、远程控制
