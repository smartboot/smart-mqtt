# 快速开始

本文将帮助你在 5 分钟内快速启动并运行 smart-mqtt。

## 环境要求

- **Java**: JDK 8 或更高版本
- **操作系统**: Linux / Windows / macOS
- **Docker**: (可选) 用于容器化部署

## 方式一：使用 Docker 部署（推荐）

smart-mqtt 提供了 Docker 镜像，这是最简单的部署方式。

### 1. 使用 docker-compose

```yaml
version: '3.8'
services:
  smart-mqtt:
    image: smartboot/smart-mqtt:latest
    container_name: smart-mqtt
    ports:
      - "1883:1883"
      - "8083:8083"
    environment:
      - TZ=Asia/Shanghai
    volumes:
      - ./data:/data
    restart: unless-stopped
```

保存为 `docker-compose.yml`，然后执行：

```bash
docker-compose up -d
```

### 2. 直接运行 Docker 容器

```bash
docker run -d \
  --name smart-mqtt \
  -p 1883:1883 \
  -p 8083:8083 \
  -e TZ=Asia/Shanghai \
  -v $(pwd)/data:/data \
  --restart unless-stopped \
  smartboot/smart-mqtt:latest
```

## 方式二：手动部署

### 1. 下载发行包

从 [Gitee Releases](https://gitee.com/smartboot/smart-mqtt/releases) 或 [GitHub Releases](https://github.com/smartboot/smart-mqtt/releases) 下载最新版本的 smart-mqtt。

```bash
wget https://gitee.com/smartboot/smart-mqtt/releases/download/v1.4.0/smart-mqtt-1.4.0.tar.gz
tar -xzf smart-mqtt-1.4.0.tar.gz
cd smart-mqtt-1.4.0
```

### 2. 启动服务

```bash
./bin/start.sh
```

Windows 用户请运行：

```bash
bin/start.bat
```

## 验证部署

### 1. 检查服务状态

```bash
# 查看进程
ps aux | grep smart-mqtt

# 查看端口监听
netstat -tlnp | grep 1883
```

### 2. 使用 MQTT 客户端测试

你可以使用任何 MQTT 客户端进行测试，例如 [MQTT X](https://mqttx.app/) 或命令行工具 `mosquitto`：

```bash
# 订阅主题
mosquitto_sub -h localhost -p 1883 -t test/topic

# 发布消息（在另一个终端）
mosquitto_pub -h localhost -p 1883 -t test/topic -m "Hello smart-mqtt"
```

## 访问 Dashboard

如果使用的是企业版，可以通过浏览器访问 Dashboard：

```
http://localhost:8083
```

## 常见问题

### Q: 启动失败，提示端口被占用？
A: 检查 1883（MQTT）或 8083（HTTP/Dashboard）端口是否已被其他服务占用。

### Q: 如何修改默认配置？
A: 编辑 `conf/smart-mqtt.properties` 文件，重启服务生效。

### Q: 如何查看日志？
A: 日志文件位于 `logs/` 目录下，可以通过 `tail -f logs/smart-mqtt.log` 实时查看。
